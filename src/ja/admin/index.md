## クイック設定

SOSI を初めて使用する管理者は、以下の4つのステップで基本設定を完了してください：

### 1. ユーザーの作成

**目的：** SOSI システムにログインできるオペレーターを作成します。

**操作：**「ユーザー一覧」で「ユーザー作成」をクリックし、システムログイン用のアカウントとパスワードを設定します。

### 2. デバイスの作成

**目的：** 接続先のリモートホスト（Windows や Linux など）を定義します。

**操作：**「デバイス一覧」でデバイスを追加し、IP アドレスを入力して正しいタイプ（RDP、SSH、または VNC）を選択します。

### 3. 資格情報の作成

**目的：** リモートホストのアカウントとパスワードを保存します（これが自動ログインの「鍵」です）。

**操作：**「資格情報設定」で資格情報を作成し、そのホストで有効なアカウントとパスワードを入力します。

### 4. 接続の認可

**目的：**「人」「扉」「鍵」を紐付けます。

**操作：**「認可デバイス一覧」に移動し、「デバイス接続認可の作成」をクリックして、認可資格情報の欄でステップ 3 で作成した資格情報を選択します。

## 機能概要

管理者コンソールは、デバイス管理、ユーザー管理、資格情報設定、接続認可、透かし保護、ドメイン認証、サイト全体設定、セキュリティ監査などのモジュールを含む、包括的なシステム管理機能を提供します。

| 機能モジュール | 説明 |
|---------|------|
| [デバイス管理](/ja/admin/devices/) | リモート接続デバイスの管理（追加、閲覧、編集、設定） |
| [ユーザー管理](/ja/admin/users/) | ユーザーアカウントの管理（作成、編集、2FA 設定） |
| [資格情報設定](/ja/admin/credentials/) | リモートホストのログイン資格情報を管理 |
| [デバイス接続認可](/ja/admin/connection-auth/) | 認可グループと接続権限の管理 |
| [ウォーターマーク設定](/ja/admin/watermark/) | 画面透かし保護メカニズムの設定 |
| [ドメイン設定](/ja/admin/domains/) | LDAP と認証ドメインの管理 |
| [サイト全体設定](/ja/admin/site-settings/) | 2FA とメールのグローバル設定 |
| [セキュリティ監査](/ja/admin/audit/) | 接続記録、ファイル転送、操作追跡 |
